# Model: Share

**Purpose:** Share link granting access to a space

**Storage:** Database `shares` table

---

## Zod Schema

```typescript
import { z } from "zod";

export const ShareSchema = z.object({
  id: z.string().uuid(),
  token: z.string(),
  spaceId: z.string().uuid(),
  userId: z.string().uuid().optional().nullable(),
  role: z.enum(["viewer", "editor", "owner", "admin"]),
  permissions: z.array(z.string()),
  label: z.string().max(100).optional(),
  expiresAt: z.string().datetime().optional().nullable(),
  createdAt: z.string().datetime(),
  revokedAt: z.string().datetime().optional().nullable(),
});

export type Share = z.infer<typeof ShareSchema>;
```

## Fields

### `id`

- **Type:** `UUID`
- **Description:** Unique identifier for this share link
- **Generated by:** Spaces Service database

### `token`

- **Type:** `string`
- **Description:** Cryptographically random token for URL
- **Generated as:** 32 random bytes, base64url encoded
- **Example:** `"Kf7Pq9RzT2mYvNcX5bS8wA1eF4gH6jK"`
- **Constraints:**
  - URL-safe
  - High entropy (256 bits)
  - Never reuse

### `spaceId`

- **Type:** `UUID`
- **Description:** ID of the space this share grants access to
- **References:** `spaces.id`

### `userId`

- **Type:** `UUID | null`
- **Description:** User who created this share
- **References:** `users.id`
- **Note:** `null` for anonymous shares (MVP)

### `role`

- **Type:** `"viewer" | "editor" | "owner" | "admin"`
- **Description:** User-facing role for this share

### `permissions`

- **Type:** `string[]`
- **Description:** System-level permissions granted by this share
- **Derived from:** Role mapping

### `label`

- **Type:** `string | undefined`
- **Description:** Human-readable label for reference
- **Example:** `"Leah's vacation link"`
- **Constraints:** Max 100 characters

### `expiresAt`

- **Type:** `Date | null`
- **Description:** When the share expires
- **Example:** `"2026-05-01T00:00:00Z"`
- **Note:** `null` = never expires

### `createdAt`

- **Type:** `Date` (ISO 8601)
- **Description:** When the share was created

### `revokedAt`

- **Type:** `Date | null`
- **Description:** When the share was revoked
- **Note:** `null` if not revoked

## Role Hierarchy

| Role | Permissions | Description | Availability |
|------|-------------|-------------|--------------|
| `viewer` | `read`, `comment` | View-only access | MVP |
| `editor` | `read`, `comment`, `edit` | Modify files | MVP |
| `owner` | `read`, `comment`, `edit`, `share` | Manage shares, full control | MVP |
| `admin` | All permissions + server admin | AI Spaces server administrator | Post-MVP |

### Role Details

#### Viewer
- View files in space
- Chat with agent
- Cannot modify anything

#### Editor
- All viewer permissions
- Edit files in space
- Agent can modify files on their behalf

#### Owner
- All editor permissions
- Create new share links
- Revoke existing share links
- Modify space configuration
- Typically the user who created the space

#### Admin (Post-MVP)
- All owner permissions
- View all spaces across all users
- Manage user accounts
- Modify any space configuration
- Server configuration
- Audit log access

## Token Security

**Token Generation:**
- 32 bytes of cryptographically random data
- Encoded as base64url (URL-safe)
- 256 bits of entropy
- Collision probability: ~0 (1 in 2^256)

**Token Validation:**
1. Extract token from URL: `/s/{spaceId}?t={token}`
2. Look up share by token in database
3. Check `revokedAt` is null
4. Check `expiresAt` is null or in future
5. Check `spaceId` matches URL parameter

## Database Schema

```sql
CREATE TABLE shares (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    token TEXT UNIQUE NOT NULL,
    space_id UUID REFERENCES spaces(id) ON DELETE CASCADE,
    user_id UUID REFERENCES users(id),
    role TEXT NOT NULL DEFAULT 'viewer',
    permissions TEXT[] NOT NULL DEFAULT ARRAY['read', 'comment'],
    label TEXT,
    expires_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW(),
    revoked_at TIMESTAMP
);

CREATE INDEX idx_shares_token ON shares(token);
CREATE INDEX idx_shares_space ON shares(space_id);
CREATE INDEX idx_shares_user ON shares(user_id);
```

## URL Format

```
https://spaces.example.com/s/{spaceId}?t={token}

Examples:
/s/550e8400...?t=Kf7Pq9Rz...     # Anonymous share link
/s/550e8400...                    # Logged-in user with owner permission
```

## Permission Mapping

```typescript
function roleToPermissions(role: Role): Permission[] {
  switch (role) {
    case 'viewer':
      return ['read', 'comment'];
    case 'editor':
      return ['read', 'comment', 'edit'];
    case 'owner':
      return ['read', 'comment', 'edit', 'share'];
    case 'admin':
      return ['read', 'comment', 'edit', 'share', 'admin'];
  }
}
```
