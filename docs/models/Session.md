# Model: Session

**Purpose:** Active WebSocket connection session

**Storage:** Database `sessions` table

---

## Zod Schema

```typescript
import { z } from "zod";

export const SessionSchema = z.object({
  id: z.string().uuid(),
  shareId: z.string().uuid(),
  agentSessionId: z.string(),
  createdAt: z.string().datetime(),
  lastActivity: z.string().datetime(),
});

export type Session = z.infer<typeof SessionSchema>;
```

---

## Fields

### `id`

- **Type:** `UUID`
- **Description:** Unique identifier for this session
- **Generated by:** Spaces Service database

### `shareId`

- **Type:** `UUID`
- **Description:** Share link used to create this session
- **References:** `shares.id`

### `agentSessionId`

- **Type:** `string`
- **Description:** Agent's internal session ID
- **Note:** Agent-adapter specific identifier

### `createdAt`

- **Type:** `Date` (ISO 8601)
- **Description:** When the session was created

### `lastActivity`

- **Type:** `Date` (ISO 8601)
- **Description:** When the session was last active
- **Updated:** On every WebSocket message

## Lifecycle

### Creation

1. Collaborator opens share URL: `/s/{spaceId}?t={token}`
2. Spaces Service validates token
3. Spaces Service creates database session record
4. Spaces Service calls agent adapter: `createSession(spaceId, context)`
5. Agent adapter returns `agentSessionId`
6. Spaces Service stores mapping: session.id → agentSessionId
7. WebSocket connection established

### Activity

1. WebSocket message received
2. Spaces Service updates `lastActivity`
3. Spaces Service forwards message to agent

### Termination

1. WebSocket disconnected
2. Spaces Service calls agent adapter: `closeSession(agentSessionId)`
3. Spaces Service deletes session record from database

## Database Schema

```sql
CREATE TABLE sessions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    share_id UUID REFERENCES shares(id) ON DELETE CASCADE,
    agent_session_id TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    last_activity TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_sessions_share ON sessions(share_id);
CREATE INDEX idx_sessions_agent ON sessions(agent_session_id);
```

## Session Management

### Active Sessions

Spaces Service tracks all active sessions:

```sql
SELECT s.*, sh.token, sp.name as space_name
FROM sessions s
JOIN shares sh ON s.share_id = sh.id
JOIN spaces sp ON sh.space_id = sp.id
WHERE s.last_activity > NOW() - INTERVAL '5 minutes';
```

### Session Cleanup

Expired sessions cleaned up periodically:

```sql
DELETE FROM sessions
WHERE last_activity < NOW() - INTERVAL '1 hour';
```

---

## Security

### Session Timeout

- Sessions expire after 1 hour of inactivity
- WebSocket ping/pong keeps session alive
- Expired sessions require reconnection

### Concurrent Sessions

- Multiple sessions allowed pershare
- Each session has unique `id` and `agentSessionId`
- Sessions don't see each other'schat history (Post-MVP feature)
