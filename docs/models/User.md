# Model: User

**Purpose:** User account with multiple authentication methods

**Storage:** Database `users` + `auth_providers` tables

**Status:** Post-MVP (MVP uses anonymous shares only)

---

## Zod Schema

```typescript
import { z } from "zod";

export const UserSchema = z.object({
  id: z.string().uuid(),
  email: z.string().email(),
  displayName: z.string().optional(),
  createdAt: z.string().datetime(),
  updatedAt: z.string().datetime(),
});

export const AuthProviderSchema = z.object({
  id: z.string().uuid(),
  userId: z.string().uuid(),
  providerType: z.enum(["email", "google", "github", "microsoft"]),
  providerUserId: z.string(),
  providerEmail: z.string().email().optional(),
  metadata: z.record(z.any()).optional(),
  createdAt: z.string().datetime(),
});

export type User = z.infer<typeof UserSchema>;
export type AuthProvider = z.infer<typeof AuthProviderSchema>;
```

## Fields

### User Fields

#### `id`

- **Type:** `UUID`
- **Description:** Unique identifier for this user
- **Generated by:** Spaces Service database

#### `email`

- **Type:** `string` (email format)
- **Description:** Primary email address (for notifications)
- **Note:** May differ from auth provider emails

#### `displayName`

- **Type:** `string | undefined`
- **Description:** User's display name
- **Source:** First auth provider or user-set

#### `createdAt`

- **Type:** `Date` (ISO 8601)
- **Description:** When the user account was created

#### `updatedAt`

- **Type:** `Date` (ISO 8601)
- **Description:** When the user account was last updated

---

### Auth Provider Fields

#### `id`

- **Type:** `UUID`
- **Description:** Unique identifier for this auth provider link

#### `userId`

- **Type:** `UUID`
- **Description:** User account this provider is linked to
- **References:** `users.id`

#### `providerType`

- **Type:** `"email" | "google" | "github" | "microsoft"`
- **Description:** Type of authentication provider

#### `providerUserId`

- **Type:** `string`
- **Description:** User ID from the auth provider
- **Example:** Google OAuth `sub`, GitHub user ID

#### `providerEmail`

- **Type:** `string | undefined`
- **Description:** Email address from the auth provider
- **Note:** May differ from `users.email`

#### `metadata`

- **Type:** `JSONB | undefined`
- **Description:** Provider-specific metadata
- **Examples:** OAuth profile data, password hash (for email)

## MVP vs Post-MVP

### MVP (v0.1)
- **No user accounts** - purely anonymous access
- Shares use `userId: null`
- Authentication is token-based only

### Post-MVP (v0.2)
- Email/password authentication
- OAuth integration (Google, GitHub, Microsoft)
- Multiple auth providers per user account
- User accounts linked to shares and spaces owned

## Database Schema

```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email TEXT UNIQUE NOT NULL,
    display_name TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE auth_providers (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    provider_type TEXT NOT NULL,
    provider_user_id TEXT NOT NULL,
    provider_email TEXT,
    metadata JSONB,
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(provider_type, provider_user_id)
);

CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_auth_providers_user ON auth_providers(user_id);
CREATE INDEX idx_auth_providers_lookup ON auth_providers(provider_type, provider_user_id);
```

## Authentication Flow (Post-MVP)

### Email/Password

1. User enters email/password in Space UI
2. Spaces Service looks up `auth_providers` where `provider_type='email'` and `provider_user_id=email`
3. Spaces Service validates password hash from `metadata.passwordHash`
4. Spaces Service creates session cookie for `user_id`
5. User can manage shares/spaces they own

### OAuth

1. User clicks "Sign in with Google"
2. Spaces Service redirects to OAuth provider
3. Provider returns OAuth token and profile
4. Spaces Service looks up `auth_providers` where `provider_type='google'` and `provider_user_id=oauth_sub`
5. If found: Log in as existing user
6. If not found: Create new user, create auth provider link
7. Spaces Service creates session cookie

### Linking Multiple Providers

After user logs in:

1. User clicks "Link GitHub account"
2. Spaces Service redirects to GitHub OAuth
3. Provider returns OAuth token
4. Spaces Service creates new `auth_provider` record linked to current `user_id`
5. User can now log in with either Google or GitHub

## Authorization

Users can have different roles per space:

```sql
CREATE TABLE user_space_roles (
    user_id UUID REFERENCES users(id),
    space_id UUID REFERENCES spaces(id),
    role TEXT NOT NULL DEFAULT 'viewer',
    created_at TIMESTAMP DEFAULT NOW(),
    PRIMARY KEY (user_id, space_id)
);
```

**Note:** Post-MVP feature.
